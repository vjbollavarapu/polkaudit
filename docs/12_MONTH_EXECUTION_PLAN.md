# PolkAudit – 12-Month Execution & Funding Plan

## Purpose of This Document

This document defines the 12-month execution roadmap and funding milestones for the PolkAudit platform.

It is intended for internal planning, treasury proposals, investor discussions, and long-term governance.

This file should be maintained in the Git repository under /docs and updated quarterly.

---

## Strategic Objectives

Over the next 12 months, PolkAudit aims to:

* Become a reliable governance transparency platform for Polkadot
* Secure Polkadot Treasury funding across two phases
* Launch a stable public MVP
* Onboard early institutional and DAO users
* Establish sustainable revenue streams

---

## Success Metrics (KPIs)

By Month 12, PolkAudit should achieve:

* 3+ active DAOs using the platform
* 1,000+ indexed governance events
* 10,000+ blocks indexed
* 95%+ system uptime
* At least 1 paid pilot client
* Minimum USD 50,000 cumulative funding

## Phase 1 – MVP & Funding Readiness (Months 1–3)

### Objectives

* Stabilize core system
* Complete documentation
* Demonstrate real on-chain data
* Prepare Polkadot Treasury proposal

### Key Deliverables

* Working indexer, backend, and dashboard
* Live data ingestion
* Compliance-ready documentation
* Demo video and screenshots
* Public GitHub repository

### Activities

* Fix indexing and persistence issues
* Improve monitoring and logging
* Complete setup, architecture, and roadmap docs
* Record demo and walkthrough
* Post Polkassembly discussion and submit Treasury referendum

### Funding Milestone

* Target: Polkadot Treasury (OpenGov), Phase 1
* Amount: USD 10,000 equivalent in DOT
* Expected Timeline: Month 2–3

### Exit Criteria

* Dashboard shows live data
* Grant application submitted
* Demo publicly accessible

---

## Phase 2 – Adoption & Reliability (Months 4–6)

* Improve system reliability

* Release reporting features
